# 可观测性总览

## 什么是可观测性

**可观测性（Observability）** 是指通过系统向外暴露的数据（信号）来推断系统内部状态的能力。这个概念源自控制理论，由 Rudolf E. Kálmán 在 1960 年提出：一个系统是可观测的，当且仅当可以通过其外部输出来唯一确定其内部状态。

在软件工程领域，可观测性意味着即使没有事先定义好"要监控什么"，你也能随时提出新问题并得到答案。这与传统监控（Monitoring）有本质区别：

| | 监控（Monitoring） | 可观测性（Observability） |
|---|---|---|
| **目标** | 跟踪已知故障模式 | 发现未知问题 |
| **数据** | 预定义指标、阈值告警 | 高基数、高维度遥测数据 |
| **提问方式** | "系统正常吗？" | "为什么会这样？" |
| **典型做法** | 配置 CPU > 90% 告警 | 任意维度下钻分析延迟变化 |

简单来说：**监控告诉你系统出了问题，可观测性帮你理解为什么出问题。**

## 三大支柱

可观测性有三类核心信号（Signals），习惯上称为"三大支柱"：

| 支柱 | 核心问题 | 数据结构 | 粒度 | 存储特点 |
|---|---|---|---|---|
| **日志（Logs）** | 发生了什么事？ | 带时间戳的不可变文本记录 | 单次事件 | 高存储、低结构化 |
| **指标（Metrics）** | 有多少、有多快？ | 带时间戳的数值（聚合量） | 时间段聚合 | 低存储、高结构化 |
| **链路追踪（Traces）** | 请求经过了哪里？ | 带因果关系的 Span 树 | 单次请求全链路 | 中等存储、高关联 |

三者不是孤立的，而是相互补充、协同工作的。它们的核心关系是：

```
     时间维度
       ↑
       │  ┌─────────────┐
       │  │   Metrics   │  ← "系统整体的快照"
       │  │   (指标)     │     回答：整体状态如何？
       │  └─────────────┘
       │
       │  ┌─────────────┐          ┌───────────────────┐
       │  │   Traces    │  ────→   │       Logs        │
       │  │  (链路追踪)  │  ←───   │      (日志)        │
       │  └─────────────┘          └───────────────────┘
       │     ↑ 因果维度              ↑ 事件细节
       │  "一次请求的完整路径"       "每个节点的详细记录"
       │
       └──────────────────────────────→ 数据量
```

- **指标**回答"整体是否健康"，适合设置告警和看趋势
- **日志**提供最详细的上下文，适合排查具体问题
- **链路追踪**连接日志与指标，展示一次请求的完整生命周期

## 核心运行模型

现代可观测性系统的数据流通常遵循以下模型：

```
┌─────────────┐     OTLP/gRPC      ┌─────────────────┐     ┌──────────┐
│   Application │  ──────────────→  │  OTel Collector  │ ──→  │  Jaeger   │ (Traces)
│   (SDK)      │      Push/Pull    │  (数据处理管道)   │ ──→  │ Prometheus│ (Metrics)
└─────────────┘                    └─────────────────┘  ──→  │  Loki     │ (Logs)
       ↑                                                    └──────────┘
  手动埋点 / 自动插桩                                          ↑ 查询与可视化
                                                        ┌──────────┐
                                                        │  Grafana  │
                                                        └──────────┘
```

关键角色：

| 组件 | 职责 |
|---|---|
| **SDK** | 在应用内生成和导出遥测数据（Logs / Metrics / Traces） |
| **Collector** | 接收、处理（过滤/采样/聚合）、路由数据到后端 |
| **后端存储** | Jaeger（追踪）、Prometheus（指标）、Loki/Elastic（日志） |
| **可视化** | Grafana 统一查询和展示三类信号 |
